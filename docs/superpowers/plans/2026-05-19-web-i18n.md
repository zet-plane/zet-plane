# Web i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a clean i18n foundation for the dashboard frontend and migrate current visible static UI chrome to English and Simplified Chinese.

**Architecture:** i18n is scoped to `apps/web` only. Locale is a user preference, not a URL concern, and product UI labels are translated while business/user-authored content remains untouched. React components translate at the UI boundary; domain, contracts, server, and CLI stay free of i18n dependencies.

**Tech Stack:** React 19, Vite, TanStack Router, TypeScript, `i18next`, `react-i18next`, browser `Intl`.

---

## Decisions

- Locale does not enter routes. Keep existing deep links stable and store language as `localStorage["zet-plane.language"]`.
- Support `en` and `zh-CN` first. English is the default and fallback language.
- Use `i18next + react-i18next` with synchronous local TS resources.
- Organize resources by product namespace: `common`, `projects`, `graph`.
- Use semantic translation keys such as `graph:canvas.loading`, not English source strings as keys.
- Use TS modules for resources. `zh-CN` resources must satisfy the English resource shape.
- Use a tiny app detector instead of `i18next-browser-languagedetector`: saved preference, then browser languages, then fallback.
- Place an `EN / 中文` segmented language switcher in page/project top chrome.
- Use `Intl.DateTimeFormat` and `Intl.RelativeTimeFormat` for dates and relative time. Translation strings provide only surrounding UI copy when needed.
- Translate UI enum labels (`blocked`, `scaffold`, `dependency`) without changing protocol, database, or URL token values.
- Development/test should surface missing translations; production should remain usable through English fallback.

## File Map

- Create `apps/web/src/i18n/index.ts`: initialize i18next, export app language helpers.
- Create `apps/web/src/i18n/languages.ts`: supported language constants, normalization, detector, persistence.
- Create `apps/web/src/i18n/resources.ts`: aggregate typed resources.
- Create `apps/web/src/i18n/i18next.d.ts`: type i18next namespaces/resources.
- Create `apps/web/src/i18n/format.ts`: date and relative-time formatting helpers.
- Create `apps/web/src/i18n/LanguageSwitcher.tsx`: reusable `EN / 中文` segmented control.
- Create `apps/web/src/i18n/*.test.ts(x)`: tests for detection, resources, formatting, and switcher behavior.
- Modify `apps/web/src/main.tsx`: initialize i18n once at app root.
- Modify `apps/web/src/router/*.tsx`: translate project list and shell chrome, render the switcher.
- Modify `apps/web/src/features/graph/components/*.tsx`: migrate visible static dashboard UI strings.
- Modify affected component tests to assert behavior/roles and the default English UI where useful.
